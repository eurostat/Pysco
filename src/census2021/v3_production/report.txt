v3: new version with corrections.

*** Changes / improvements / corrections:
- value set to -8888 when not available because confidential and to -9999 when not available, for other undocumented reason. Consequently, no "CI" attribute. And... no more confidentiality / value consistency issue.
- added a new column called 'cntr_id' with reporting countries
- when T=0 and all others are not available, then all are set to 0. Applies to BE, and FR (EMP) and DE (EMP)
- EL reported confidential value (column SPECIAL_VALUE) as "notApplicable" while other country leave it blank. Consider that as black.
- There were 183 cells that are described in country data that have been lost in v2
11 are for non allocated data for FR, IT, FI, BG, EL, SE, DK, NO, BE, LV, LU, SI
The others are 0 population cells in DE, DK, EL, NL, PL, PT
This has been corrected. unalocated population: it is now kept in the disseminated data. For GPKG, features without geometries with id=cc_unallocated
- simplified formats: CSV and parquet are no longer data item based but cell based. A single geotiff with several bands.

- cases of values provided for confidential flag. 611 cells in LU (466 cells) and SK (145 cells)
in such case, value (not ?) disseminated.
Maybe it could and the confidential value is a mistake (for LU (high values) but not for SK).


- land_area:
rounding issues. 0.5529999999999 instead of 0.553 (for CRS3035RES1000mN2288000E3891000)
NA in country data replaced with 0. (RO, NL, DK, FR, coast)
This has been corrected.

For pixels shared between countries, it is assumed each country reports the land_area of its part of the cell. The sum of the land area contributions from the countries sharing a cell should thus in theory remain equal or below 1.
But when summing these contributions, many shared cells result in values above 1. In such case, the land-surface value is ceiled to 1.


- POPULATED attribute is problematic in input data. Should be "O" or "1". Value "" present, considered as "0".
Many times (for almost all cells - 4.5M out of ) not compliant with the rules:
            # data items on total population with an observed value other than ‘0’ shall be marked with the flag ‘populated’;
            # and (c) data items on total population with an observed value ‘0’ shall not be marked with the flag ‘populated’. 
            # The flag ‘populated’ shall be applicable only to ‘total population’
Rule chosen: provided value ignored and set to 1 as soon as an observed value is positive or confidential

In v2, 32573 cells were populated is 1 but should be 0. 326 cells where it is 0 but should be 1
This has been corrected.
